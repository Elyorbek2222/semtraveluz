# Урок 3: Анатомия SEO-оптимизированного контента

Каждая высокоранжированная страница следует определённой структуре. Понимание этой анатомии необходимо перед автоматизацией — AI должен следовать тем же правилам.

## Иерархия заголовков

### H1 — Главный заголовок
- **Только один на странице**
- Содержит главное ключевое слово
- Это заголовок страницы в целом

### H2 — Основные разделы
- **3-6 на статью** (оптимально)
- Включают ключевые слова естественно
- Разбивают контент на логические блоки

### H3 — Подразделы
- Находятся внутри H2 разделов
- Добавляют глубину и структуру
- Помогают читателям навигировать

## Стратегическое размещение ключевых слов

### Title-тег
- **Самый важный on-page сигнал**
- 50-60 символов (максимум)
- Ключевое слово ближе к началу
- Пример: `SEO-автоматизация с AI: Полное руководство 2026`

### Meta Description
- **120-160 символов**
- Влияет на CTR (Click-Through Rate)
- Должна содержать основное ключевое слово
- Должна включать призыв к действию
- Пример: `Создайте полную систему SEO-автоматизации с AI. От ключевых слов до авто-публикации. Кейс: с 800 до 12К посетителей.`

### Первый абзац (Body Text)
- Ключевое слово в первых 100 словах
- Помогает Google понять тему страницы

### Заголовки H2
- Минимум 2 должны содержать ключевые слова
- Естественное включение
- Без переспама

### Alt-текст изображений
- Описывайте с релевантными ключевыми словами
- Помогает скринридерам и Google
- Должен быть описательным, не спамным

## Важные мета-теги — Примеры

```html
<title>SEO-автоматизация с AI: Полное руководство 2026</title>
<meta name="description" content="Создайте полную систему SEO-автоматизации с AI. От ключевых слов до авто-публикации. Кейс: с 800 до 12К посетителей.">
<meta name="keywords" content="SEO автоматизация, AI контент, автопилот">
<link rel="canonical" href="https://example.com/page">
```

## Внутренняя перелинковка (Internal Linking)

- **Каждая статья должна содержать 2-3 внутренние ссылки**
- Ссылки на другие страницы сайта
- Помогает Googlebot обнаруживать контент
- Распределяет авторитет (PageRank) по сайту
- Улучшает пользовательский опыт

### Правила:
- Ссылайтесь на релевантные страницы
- Используйте якорный текст с ключевыми словами
- Не более 3-4 ссылок в одной статье (не спамить)

## FAQ-секции

### Зачем нужны:
- Помогает попасть в **featured snippets** (позиция 0)
- Охватывает длиннохвостые запросы
- Увеличивает время на странице
- Улучшает пользовательский опыт

### Структура:
```html
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "Вопрос?",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "Ответ..."
      }
    }
  ]
}
</script>
```

## Плотность ключевых слов

- **Оптимальная плотность: 1-2%**
- **Максимум: 2.5%** (больше Google может счесть переспамом)
- Рассчитывается: (количество ключевых слов / всё слова в тексте) × 100

### Пример:
- Статья из 1000 слов
- Ключевое слово должно повторяться 10-20 раз (1-2%)
- Максимум 25 раз (2.5%)
