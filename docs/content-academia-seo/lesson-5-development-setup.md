# Урок 5: Настройка среды разработки

Прежде чем писать код SEO-автоматизации, нужна правильно настроенная среда. Этот урок проведёт вас через каждый шаг.

## Шаг 1: Установите Node.js 22+

```bash
node --version    # Должно показать v22.x или выше
npm --version     # Идёт вместе с Node.js
```

### Что такое Node.js?
- **Node.js** — это JavaScript runtime для выполнения кода вне браузера
- **npm** — package manager (менеджер пакетов) для Node.js
- **npm install** — устанавливает библиотеки

## Шаг 2: Установите Claude Code

```bash
npm install -g @anthropic-ai/claude-code
claude --version
```

### Что такое Claude Code?
- CLI-инструмент для работы с AI
- Интеграция с Anthropic API
- Позволяет автоматизировать разработку

## Шаг 3: Создайте проект

```bash
mkdir seo-automation && cd seo-automation
npm init -y
```

### Что произойдёт:
- Создастся папка `seo-automation`
- Файл `package.json` с конфигурацией проекта
- `-y` означает: ответить "yes" на все вопросы

## Шаг 4: Установите зависимости

```bash
npm install axios cheerio
npm install -D typescript tsx
```

### Что устанавливаем:

#### axios
- **HTTP-клиент** для загрузки страниц
- Используется для вызова API
- Заменяет встроенный `fetch`

#### cheerio
- **HTML-парсер** (как jQuery для Node.js)
- Позволяет извлекать данные из HTML
- Быстрее и проще чем DOM API

#### typescript (dev)
- **Типизация** для JavaScript
- Помогает избежать ошибок
- Рекомендуется для production-кода

#### tsx (dev)
- **TypeScript executor** — запускает TypeScript напрямую
- Не требует компиляции

## Шаг 5: Получите API-ключ Claude

1. Перейдите на **console.anthropic.com**
2. Создайте аккаунт или войдите
3. Перейдите в **API Keys**
4. Создайте новый ключ и скопируйте его

### Как это выглядит:
```
sk-ant-abc123defg456hij789klmnop...
```

## Шаг 6: Создайте .env

```bash
# .env
ANTHROPIC_API_KEY=sk-ant-your-key-here
```

### Важно:
- Добавьте `.env` в `.gitignore`
- Никогда не коммитьте API-ключи
- Это ваш личный ключ доступа

### .gitignore:
```
.env
.env.local
node_modules/
.DS_Store
*.log
```

## Пусть Claude Code всё настроит

Вместо ручной настройки, используйте Claude Code:

```bash
claude
```

Затем дайте команду:

```
Настрой Node.js проект для SEO-автоматизации. 
Создай package.json с type:module, 
установи axios и cheerio. 
Создай папку src/ с index.js, 
который выводит "SEO Automation System Ready". 
Проверь что работает.
```

### Что Claude Code сделает:
1. ✅ Создаст `package.json` с нужной конфигурацией
2. ✅ Установит `axios` и `cheerio`
3. ✅ Создаст структуру папок
4. ✅ Напишет простой тестовый скрипт
5. ✅ Запустит и проверит что работает

## Структура проекта

```
seo-automation/
├── src/                  # Исходный код
│   ├── index.js
│   ├── crawler.js
│   ├── generator.js
│   └── validator.js
├── data/                 # Выходные данные
│   ├── articles/
│   └── reports/
├── config/               # Конфигурация
│   └── settings.js
├── .env                  # API-ключи (в .gitignore)
├── .gitignore
├── package.json
└── README.md
```

### Назначение папок:
- **src/** — весь ваш код
- **data/** — сгенерированные статьи и отчёты
- **config/** — настройки приложения
- **.env** — секретные ключи

## Проверка готовности

```bash
# Проверьте Node.js
node --version

# Проверьте npm
npm --version

# Проверьте установленные пакеты
npm list

# Запустите тестовый скрипт
node src/index.js
```

### Результат должен быть:
```
SEO Automation System Ready ✓
```

## Совет для организации проекта

**Держите проект чистым:**
- `src/` — только исходный код
- `data/` — только выходные файлы
- `config/` — только конфигурация
- Никаких случайных файлов в корне

## Почему чистая среда важна

**Чистая среда разработки предотвращает 90% проблем с отладкой:**
- 🟢 Легче найти ошибки
- 🟢 Проще поделиться кодом
- 🟢 Больше шансов, что код будет работать на других машинах
- 🟢 IDE и редакторы работают быстрее

## Следующий шаг

После настройки среды вы готовы к первому модулю — созданию системы выбора темы и подбора ключевых слов.

Если что-то не работает:
1. Проверьте версию Node.js (должна быть 22+)
2. Удалите `node_modules/` и переустановите: `npm install`
3. Используйте Claude Code: просто опишите проблему
