# Модуль 2, Урок 5: Планирование контента и выбор тем

Автоматическая генерация контента хороша настолько, насколько хороши подобранные темы. В этом уроке вы создадите умную систему выбора тем.

## Ротация категорий

**Система автоматически выбирает категорию с наименьшим числом постов за 30 дней.**

Это гарантирует **сбалансированный контент** по всем категориям.

### Пример:

```
Категории:       Постов (30д):   Выбрана:
┌─────────────────────────────────────────┐
│ Технологии       8 постов               │
│ Бизнес           3 поста        ← ✅   │
│ Маркетинг        5 постов               │
│ Дизайн           7 постов               │
│ Автоматизация    4 поста               │
└─────────────────────────────────────────┘
```

**Логика:** Выбирается "Бизнес" (наименьшее число = 3)

### Результат:
- ✅ Нет перекоса в сторону популярных категорий
- ✅ Читатели видят разнообразный контент
- ✅ Google предпочитает сбалансированные сайты

## Предотвращение дубликатов

**Перед генерацией система проверяет:**

### Проверка 1: Заголовок
```
❌ Не генерируй если заголовок существует
Пример: "SEO в 2026" уже есть → пропусти
```

### Проверка 2: Ключевые слова
```
❌ Не генерируй если ключевые слова совпадают 80%+
Пример: 
- Статья 1: "SEO, автоматизация, инструменты"
- Статья 2: "SEO, автоматизация" → 67% совпадение ✅ OK
- Статья 3: "SEO, автоматизация, инструменты, платформа" → 80%+ совпадение ❌ SKIP
```

### Проверка 3: Временная давность
```
❌ Не генерируй если тема была за последние 90 дней
Пример:
- Написана статья на "Web Development" 45 дней назад
- Попытка написать ещё одну → 90 дней не прошло ❌ SKIP
```

Затем дайте команду:

```
Создай Node.js скрипт topic-selector.js, который:
1. Читает категории из JSON-конфига
2. Считает статьи по категориям за последние 30 дней
3. Выбирает категорию с наименьшим числом
4. Генерирует 3 альтернативные темы через Claude API
5. Проверяет каждую на дубликаты (заголовок, ключевые слова, временная давность)
6. Возвращает лучшую уникальную тему
7. Сохраняет план в JSON в папку content-plans/
8. Выводит красивый отчёт с выбранной темой
```

## Логика селектора (пошагово)

### Шаг 1: Прочитать конфиг
```javascript
const categories = JSON.parse(fs.readFileSync('categories.json'));
// Результат: [{id: "cat-1", name: "Business"}, ...]
```

### Шаг 2: Подсчитать статьи
```javascript
const stats = categories.map(cat => ({
  id: cat.id,
  name: cat.name,
  count: countArticlesInCategory(cat.id, 30) // последние 30 дней
}));
```

### Шаг 3: Выбрать категорию
```javascript
const selected = stats.reduce((prev, curr) => 
  curr.count < prev.count ? curr : prev
);
// Результат: категория с минимальным числом статей
```

### Шаг 4: Сгенерировать темы
```javascript
const themes = await generateThemesViaClaude(
  selected.name,
  { temperature: 0.7, count: 3 }
);
// Результат: 3 альтернативные темы
```

### Шаг 5: Проверить дубликаты
```javascript
themes.forEach(theme => {
  const isDuplicate = 
    checkTitleExists(theme.title) ||
    checkKeywordOverlap(theme.keywords, 80) ||
    checkTimeSinceLastPost(theme.topic, 90);
  
  if (!isDuplicate) {
    best_theme = theme;
  }
});
```

### Шаг 6: Сохранить план
```javascript
const plan = {
  id: generateId(),
  topic: theme,
  categoryId: selected.id,
  status: "planned",
  scheduledFor: new Date()
};

fs.writeFileSync(`content-plans/${plan.id}.json`, JSON.stringify(plan));
```
